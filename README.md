# Noita-like Pixel Engine (C++)

Кроссплатформенный движок для симуляции частиц (клеточных автоматов) в стиле игры Noita.

## Структура проекта
- `core/`: Ядро симуляции. Компилируется в динамическую библиотеку (DLL/SO).
- `platforms/desktop/`: Пример приложения на Raylib для Windows/Linux.
- `platforms/android/`: (Заготовка) Для сборки под Android через NDK.

## Как собрать (Desktop)

### Зависимости
1. **CMake** (3.10+)
2. **C++ Compiler** (C++17)
3. **Raylib** (Для графики)

### Сборка
```bash
mkdir build && cd build
cmake ..
make
```

## Как собрать для Android
Для Android используйте **Android NDK**. Ядро компилируется в `.so` файл, который подключается через JNI к вашему Android-проекту.
Raylib также поддерживает Android, вы можете использовать `NativeActivity`.

## Особенности
- **Sand (Песок):** Падает вниз, образует кучи.
- **Water (Вода):** Течет вниз и растекается в стороны.
- **Fire (Огонь):** Поднимается вверх и случайно затухает.
- **Wall (Стена):** Статичные пиксели, которые нельзя сдвинуть.

## Планы по развитию
1. **Многопоточность:** Разбить мир на чанки для параллельного обновления.
2. **Физика тел:** Интеграция с Box2D для взаимодействия твердых тел с пиксельным миром.
3. **Шейдеры:** Перенос рендеринга и, возможно, части симуляции на GPU (Compute Shaders).
